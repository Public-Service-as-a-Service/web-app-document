# UI Components

This project uses [shadcn/ui](https://ui.shadcn.com) as its component system. Components are not installed as a package — they live as source files in `src/components/ui/` and are fully owned by the project.

## Available components

### shadcn primitives

These are generated by `npx shadcn@latest add <name>` and can be edited directly.

| Component    | File             | Purpose                                                                                                                                                     |
| ------------ | ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Button`     | `button.tsx`     | Buttons with variants: `default`, `secondary`, `destructive`, `outline`, `ghost`, `link`. Sizes: `default`, `xs`, `sm`, `lg`, `icon`, `icon-sm`, `icon-lg`. |
| `Input`      | `input.tsx`      | Text input field                                                                                                                                            |
| `Textarea`   | `textarea.tsx`   | Multi-line text input                                                                                                                                       |
| `Select`     | `select.tsx`     | Dropdown select (compound: `Select > SelectTrigger > SelectValue` + `SelectContent > SelectItem`)                                                           |
| `Switch`     | `switch.tsx`     | Toggle switch                                                                                                                                               |
| `Tabs`       | `tabs.tsx`       | Tabbed interface (compound: `Tabs > TabsList > TabsTrigger` + `TabsContent`)                                                                                |
| `Dialog`     | `dialog.tsx`     | Modal dialog (compound: `Dialog > DialogContent > DialogHeader/Title/Description/Footer`)                                                                   |
| `Label`      | `label.tsx`      | Form label                                                                                                                                                  |
| `Pagination` | `pagination.tsx` | Pagination primitives (presentational only)                                                                                                                 |
| `Badge`      | `badge.tsx`      | Status badges                                                                                                                                               |
| `Skeleton`   | `skeleton.tsx`   | Loading placeholder                                                                                                                                         |
| `Separator`  | `separator.tsx`  | Visual divider                                                                                                                                              |
| `Tooltip`    | `tooltip.tsx`    | Hover tooltip                                                                                                                                               |

### Custom components

Built on top of shadcn primitives for app-specific needs.

| Component       | File                 | Purpose                                                                                  |
| --------------- | -------------------- | ---------------------------------------------------------------------------------------- |
| `ConfirmDialog` | `confirm-dialog.tsx` | Confirmation dialog for destructive actions. Wraps `Dialog` with confirm/cancel buttons. |
| `SearchInput`   | `search-input.tsx`   | Search field with icon, clear button, and `onSearch` callback.                           |
| `PaginationNav` | `pagination-nav.tsx` | Stateful pagination with page numbers, prev/next. Wraps `Pagination`.                    |

## Usage patterns

### Importing

```tsx
import { Button } from '@components/ui/button';
import { Input } from '@components/ui/input';
import {
  Dialog,
  DialogContent,
  DialogHeader,
  DialogTitle,
  DialogFooter,
} from '@components/ui/dialog';
```

### Class merging with `cn()`

Use `cn()` from `@lib/utils` to merge Tailwind classes conditionally:

```tsx
import { cn } from '@lib/utils';

<div className={cn('rounded-lg p-4', isActive && 'bg-primary/10')} />;
```

### Button

```tsx
// Primary action
<Button onClick={handleSave}>Save</Button>

// Secondary
<Button variant="secondary">Cancel</Button>

// Ghost (subtle)
<Button variant="ghost" size="sm">Back</Button>

// Destructive
<Button variant="destructive">Delete</Button>

// With icon
<Button>
  <Plus className="mr-2 h-4 w-4" />
  Create
</Button>

// Icon-only
<Button variant="ghost" size="icon" aria-label="Delete">
  <Trash2 className="h-4 w-4" />
</Button>

// Loading state
<Button disabled={loading}>
  {loading && <Loader2 className="mr-2 h-4 w-4 animate-spin" />}
  Save
</Button>
```

### Form fields

```tsx
// Label + Input
<div className="space-y-2">
  <Label htmlFor="name">
    Name <span className="text-destructive">*</span>
  </Label>
  <Input id="name" value={value} onChange={(e) => setValue(e.target.value)} />
</div>

// Select
<Select value={value} onValueChange={setValue}>
  <SelectTrigger>
    <SelectValue placeholder="Choose..." />
  </SelectTrigger>
  <SelectContent>
    <SelectItem value="a">Option A</SelectItem>
    <SelectItem value="b">Option B</SelectItem>
  </SelectContent>
</Select>

// Switch with label
<div className="flex items-center gap-2">
  <Switch id="toggle" checked={checked} onCheckedChange={setChecked} />
  <Label htmlFor="toggle">Enable feature</Label>
</div>
```

### Dialogs

```tsx
// Controlled dialog
<Dialog open={isOpen} onOpenChange={setIsOpen}>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>Edit item</DialogTitle>
    </DialogHeader>
    {/* Form content */}
    <DialogFooter>
      <Button variant="secondary" onClick={() => setIsOpen(false)}>Cancel</Button>
      <Button onClick={handleSave}>Save</Button>
    </DialogFooter>
  </DialogContent>
</Dialog>

// Confirm dialog (for delete/destructive actions)
<ConfirmDialog
  open={showConfirm}
  onOpenChange={setShowConfirm}
  title="Delete this item?"
  description="This action cannot be undone."
  confirmLabel="Delete"
  cancelLabel="Cancel"
  variant="destructive"
  onConfirm={handleDelete}
/>
```

### Tabs

```tsx
<Tabs defaultValue="details">
  <TabsList>
    <TabsTrigger value="details">Details</TabsTrigger>
    <TabsTrigger value="history">History</TabsTrigger>
  </TabsList>
  <TabsContent value="details">...</TabsContent>
  <TabsContent value="history">...</TabsContent>
</Tabs>
```

### Loading states

Use `Loader2` from lucide-react with `animate-spin`:

```tsx
import { Loader2 } from 'lucide-react';

// Standalone spinner
<Loader2 className="h-8 w-8 animate-spin text-muted-foreground" />

// Inside a button
<Button disabled>
  <Loader2 className="mr-2 h-4 w-4 animate-spin" />
  Saving...
</Button>
```

## Adding new shadcn components

Browse available components at [ui.shadcn.com](https://ui.shadcn.com/docs/components).

```bash
npx shadcn@latest add dropdown-menu
npx shadcn@latest add toast
npx shadcn@latest add sheet
npx shadcn@latest add card
```

After adding, fix the import path in the generated file:

```diff
- import { cn } from "@components/lib/utils"
+ import { cn } from "@lib/utils"
```

## Customizing existing components

shadcn components are source files you own. To add a new button variant:

```tsx
// Edit src/components/ui/button.tsx
const buttonVariants = cva('...', {
  variants: {
    variant: {
      // ...existing variants
      warning: 'bg-amber-500 text-white hover:bg-amber-600',
    },
  },
});
```

## Building custom components

When creating app-specific components, compose from shadcn primitives:

```tsx
// src/components/ui/my-component.tsx
import { Button } from '@components/ui/button';
import { Dialog, DialogContent, ... } from '@components/ui/dialog';

export function MyComponent({ ... }) {
  // Combine shadcn primitives with your own logic
}
```

Place reusable UI components in `src/components/ui/`. Place feature-specific components in `src/components/<feature>/`.

## Design tokens

All colors come from CSS variables defined in `src/styles/globals.css`. Use semantic names:

| Token                   | Usage                                                |
| ----------------------- | ---------------------------------------------------- |
| `text-foreground`       | Primary text                                         |
| `text-muted-foreground` | Secondary/subtle text                                |
| `bg-background`         | Page background                                      |
| `bg-card`               | Card/panel background                                |
| `bg-muted`              | Subtle background (table headers, input backgrounds) |
| `bg-accent`             | Hover/active states                                  |
| `bg-primary`            | Brand color (buttons, active nav)                    |
| `bg-primary/10`         | Tinted brand background                              |
| `border-border`         | Default borders                                      |
| `text-destructive`      | Error/danger text                                    |
| `bg-destructive/10`     | Error background                                     |

These tokens automatically adapt to light/dark mode and tenant branding.

## Multi-tenant theming

Colors are overridable per tenant via `NEXT_PUBLIC_TENANT_ID`. See `src/config/tenants/` for configuration. Access tenant data (logo, name) in components:

```tsx
import { useTenant } from '@components/tenant-provider/tenant-provider';

const tenant = useTenant();
// tenant.appName, tenant.logo.src, etc.
```
